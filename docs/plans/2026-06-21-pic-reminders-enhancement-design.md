# Design Specification: PIC Reminders and Gratitude Messages Enhancement

This document outlines the design specification for enhancing the group reminder and completion/gratitude messages generated by the PIC from the dashboard.

---

## 1. Requirements

### A. Pre-Registration invitation / Reminder (Status: `Open` / `Closed`)
When the campaign is not yet finalized, the PIC can copy a group invite link.
* **Goal**: Encourage colleagues to register for the gift pool.
* **Details**: Include Target Name, Reason (if set), Deadline, the list of currently joined participants, and a registration web URL.

### B. Post-Finalization Group Billing Reminder (Status: `Finalized` and unpaid donors exist)
When the bills have been calculated but some donors have not verified their payments.
* **Goal**: Remind unpaid donors to transfer their contributions.
* **Details**:
  * List all unpaid donors. (Indicate `Nominal Bebas` next to their name if they pledged a custom amount).
  * Output pro-rata math breakdowns.
  * **Case 1 (No Custom Pledges)**: Total Gift, Total Donors, Split amount.
  * **Case 2 (With Custom Pledges)**: Total Gift, Total Donors, count of custom donors and their total sum, remaining target amount, split amount for standard donors, and standard donor count.
  * Transfer bank details.
  * Web link directly containing the campaign reference parameter (`?c=CampaignID`).

### C. Campaign Completion / Gratitude Message (Status: `Finalized` and all donors verified paid)
When all payments have been collected and verified by the PIC.
* **Goal**: Share the final success summary and express gratitude to the participants.
* **Details**:
  * Confirm the campaign is complete and verified.
  * List total gift amount.
  * Show total custom pledges sum and count of custom donors (without individual names to respect privacy).
  * List all participant names as a gratitude roll.

---

## 2. Technical Design

### A. Backend Changes (`Code.js`)

We will refactor/extend `getReminderInfo(picToken)` to return the correct text message based on the campaign status.

#### `getReminderInfo(picToken)` Logic:
1. Verify PIC token validation.
2. Load Campaign and Donor details.
3. Identify Web App Base URL:
   * Use `ScriptApp.getService().getUrl()` dynamically.
4. Route message generation:
   * **Case `Open` or `Closed`**:
     * Call helper `generatePreRegistrationMessage_(campaign, donors, webUrl)`.
   * **Case `Finalized`**:
     * If there are unpaid donors:
       * Call helper `generateGroupBillingReminder_(campaign, donors, webUrl)`.
     * If all donors are paid:
       * Call helper `generateGratitudeMessage_(campaign, donors)`.

---

### B. Frontend Changes (`Index.html`)

#### 1. UI Actions in `renderPicActions(detail)`
Update the "Share" box in the PIC Dashboard:
* If status is `Open` or `Closed`:
  * Title: "Bagikan Undangan Patungan:"
  * Button: `📋 Copy Undangan Patungan` (calls `copyPicGroupReminder()`)
* If status is `Finalized`:
  * If achieved (all verified paid):
    * Title: "Target Terkumpul! 🎉"
    * Button: `📋 Copy Laporan Selesai (Gratitude)` (calls `copyPicGroupReminder()`)
  * If not achieved (unpaid donors remain):
    * Title: "Tagihan Patungan (Grup):"
    * Button: `📋 Copy Tagihan Patungan (Grup)` (calls `copyPicGroupReminder()`)

#### 2. Action Handler `copyPicGroupReminder()`
* Add JS function `copyPicGroupReminder()`:
  * Calls backend `getReminderInfo(currentToken())`.
  * Copies the returned text into the clipboard.
  * Shows toast confirmation.
